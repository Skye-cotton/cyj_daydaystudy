在ES2015之前，只有var

var会造成变量污染



var 变量提升

let 暂时性死区

const 变量必须在声明时赋值,不能重新赋值。

#### 总结

- var可以重复声明

  var x=10;

  var x=6;

  console.log(x);  // 6

- let 不能重新声明var变量

  var x=10;

  let x=6;  // 不允许

  {

  var x=10;

  let x=6;  // 不允许

  }

- var不能重新声明let变量

- let 不能重复声明，但是可以在不同域中重新声明

  ```
  let x = 6;       // 允许
  
  {
    let x = 7;   // 允许
  }
  
  {
    let x = 8;   // 允许
  }
  ```

`var`、`let`、`const`三者区别可以围绕下面五点展开：

- 变量提升
- 暂时性死区
- 块级作用域
- 重复声明
- 修改声明的变量
- 使用
